## Estrutura do site

# Regras a se seguir:

- O site deve ser feito usando JavaScript, HTML e CSS, usando o padrão de Bootstrap como base;
- Não se deve usar frameworks ou bibliotecas desnecessárias que podem atrapalhar a usabilidade do site;
- Seguir as regras de IHC e IHD estabelecidas no arquivo "objetivos_0.md";
- Também seguir outras possíveis regras citadas em outros arquivos .md desse projeto que foram deixadas de fora do arquivo de objetivos;
- Respeitar as personas apresentadas no arquivo "User_Stories.md", lembrando que o site é voltado para elas;
- Trabalhar a responsividade do site para que ele possa ser usado em diferentes dispositivos;
- Trabalhar a acessibilidade do site para que ele possa ser usado por pessoas com deficiência visual e/ou analfabetas digitais;
- Os dados serão todos simulados usando como base a cidade de Bandeirantes - Paraná, ou seja, apenas os endereços devem ser reais, os pontos de coletas devem ser criados, assim como os dados apresentados no dashboard. Lembrando de não usar propriedades privadas para a criação desses endereços e nem números reais de telefone;
- Hierarquia visual e de pastas bem definida.
